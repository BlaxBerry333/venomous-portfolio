## 課題と背景

以前の練習プロジェクトでは同じ機能や UI を繰り返し実装することが多く、時間の浪費や標準化不足が課題でした。

また、Material UI や Ant Design など既存ライブラリは機能豊富ながらもカスタマイズ性に難があり、自分のニーズに最適化しづらいという問題がありました。

そこで、ゼロから独自の UI ライブラリを構築することで「開発効率」と「柔軟性」を両立。さらにドキュメント化や CI/CD を含め、単なるコンポーネント実装を超えた「再利用可能なフロントエンド基盤」として整備しました。

#### スタイル設計

- 複数テーマ切替機能を実装し、カラースキームやタイポグラフィを Design Token として管理。これにより、デザインの一貫性と拡張性を確保
- inline-style と className ( Tailwind CSS スタイル ) の両方に対応し、利用者が最適な方法でスタイルを上書きできる高いカスタマイズ性を実現
- **Framer Motion** を導入し、宣言的なアニメーションを実装。ユーザー体験を向上させるインタラクティブなコンポーネントを提供

#### モジュール最適化

- Vite ESM サポートを活用し、マルチエントリビルド ( multi-entry / multi-output ) を採用。これにより、利用者は必要なモジュールだけをインポートでき ( オンデマンド読込 )、Tree Shaking によって最終的なバンドルサイズを最小化

#### 開発プロセス

- **GitHub Actions** を用いた CI/CD
- **GitHub Pages** に **Storybook** インタラクティブページとドキュメントを自動デプロイ
- **Husky + lint-staged + Prettier + ESLint + cspell** などを活用してコード規約と品質を維持

## 得られた知見

- コンポーネント実装に留まらず、Design Token 設計、ドキュメント化、CI/CD パイプラインまで含めたフロントエンド基盤のデザインシステム全体をゼロから構築する能力を証明
- 大規模な開発チームにおいてデザインと実装の一貫性を保ち、開発効率を最大化するためのフロントエンド基盤構築に直接活かすことができます。再利用性と拡張性を考慮したアーキテクチャ設計能力を示した
