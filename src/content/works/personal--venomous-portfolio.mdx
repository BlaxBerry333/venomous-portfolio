## 課題と背景

自分の制作物を体系的に公開・管理するために、ポートフォリオサイトを構築しました。

構築にあたっては Next.js、Nuxt.js、Gatsby.js、Vite.js なども検討したが、個人開発における「開発体験」「保守性・拡張性」「パフォーマンス」のバランスを踏まえ、最終的に Astro を選択した。

#### Astro 採用の主な理由

- **Islands Architecture** により静的 HTML に必要最小限の JavaScript のみを読み込み、作品ページの表示速度と柔軟性を両立
- **Content Collections** により「コンテンツ as データ」の考え方を採用し、**Markdown/MDX + Zod** で型安全な作品データの編集・管理が可能
- Astro は React / Vue / Svelte などのコンポーネントを柔軟に組み合わせられるため、UI 拡張の自由度が高い

#### 開発プロセス

- **GitHub Actions** による CI/CD
- **GitHub Pages** への自動デプロイ
- **Husky + Prettier + ESLint** などでコード規約と品質を維持
